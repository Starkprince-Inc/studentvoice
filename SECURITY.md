# Security policy

Do not open public issues for vulnerabilities, exposed source information, unredacted evidence, authentication bypasses, or identity/publication-gate failures. Use GitHub private vulnerability reporting after the repository is published, or contact the security team through a separately published encrypted channel.

Include affected version, reproduction steps using synthetic data, impact, and any known mitigation. Do not access, download, alter, or retain evidence beyond the minimum necessary to demonstrate the issue.

Supported versions: the latest tagged release and the default branch. Critical issues involving source exposure, unauthorized originals, publication bypass, identity disclosure, key compromise, or audit tampering trigger the incident-response process immediately.
